# Деплой демо: бэк (WordPress) + Mini App в Telegram

## Вариант с ngrok (бесплатно, для демо)

Ngrok даёт временный публичный URL на твой локальный WordPress. Для демо этого достаточно. Минус: при каждом новом запуске ngrok URL меняется (в бесплатной версии), и нужно будет один раз обновить конфиг фронта.

---

## Шаг 1. Поднять WordPress локально

1. Запусти свой WordPress с плагином HR Ecosystem (XAMPP, OpenServer, Local by Flywheel или docker — как у тебя уже настроено).
2. Убедись, что сайт открывается в браузере по адресу вида `http://hr-ecosystem.local` или `http://localhost/wordpress` и что плагин активен.
3. Включи в настройках WordPress (Настройки → Общие) опцию «Любой может зарегистрироваться», если нужно автосоздание пользователей из Telegram (или оставь как есть — пользователи создаются через `POST /auth`).

---

## Шаг 2. Установить и запустить ngrok

1. Скачай ngrok: https://ngrok.com/download (или `choco install ngrok` / `winget install ngrok` в Windows).
2. Зарегистрируйся на https://ngrok.com и в кабинете скопируй свой **authtoken**.
3. В терминале выполни:
   ```bash
   ngrok config add-authtoken ВАШ_ТОКЕН
   ```
4. Запусти туннель на порт, где крутится WordPress. Например, если WP на порту 80:
   ```bash
   ngrok http 80
   ```
   Если WP на порту 8080:
   ```bash
   ngrok http 8080
   ```
5. В выводе ngrok появится строка **Forwarding** вида:
   ```
   https://abc123def.ngrok-free.app -> http://localhost:80
   ```
   Скопируй этот **https-URL** (например `https://abc123def.ngrok-free.app`) — это и есть временный адрес твоего бэка.

---

## Шаг 3. Настроить WordPress под ngrok (если нужно)

1. В админке WordPress: **Настройки → Общие**.
2. Поля **«Адрес WordPress (URL)»** и **«Адрес сайта (URL)»** временно поставь свой ngrok-URL (например `https://abc123def.ngrok-free.app`). Сохрани.
3. После демо можно вернуть обратно локальный адрес.

Так API будет отдавать правильные ссылки, и запросы с Mini App пойдут на ngrok.

---

## Шаг 4. Фронт (Mini App) — Vercel

1. Фронт уже задеплоен на Vercel (например `https://hr-front-mu.vercel.app`).
2. В репозитории открой **mini-app/js/config.js**.
3. Поставь:
   - **API_BASE_URL** = твой ngrok-URL **без** слэша в конце, например: `'https://abc123def.ngrok-free.app'`
   - **DEV_TOKEN** оставь пустой строкой `''` для демо из Telegram.
4. Закоммить изменения и дождаться пересборки на Vercel (или сделай новый деплой).

Каждый раз после нового запуска ngrok (новый URL) нужно повторить п.3 и заново задеплоить фронт.

---

## Шаг 5. Создать бота и подключить Mini App в Telegram

1. Открой Telegram, найди **@BotFather**.
2. Отправь команду `/newbot`, придумай имя и username бота (например `HR Demo Bot` и `hr_demo_xxx_bot`). BotFather пришлёт **токен бота** — сохрани его.
3. В WordPress в настройках плагина HR (если есть поле для токена бота) — вставь этот токен и сохрани.
4. В BotFather отправь команду **/newapp** или через меню выбери создание/редактирование Mini App для своего бота.
5. Когда попросит **Mini App URL**, введи URL фронта на Vercel, например:
   ```
   https://hr-front-mu.vercel.app
   ```
   Без слэша в конце. Подтверди.
6. При необходимости задай название и описание Mini App (для демо можно любое).

После этого в профиле бота или в меню бота появится кнопка открытия Mini App.

---

## Шаг 6. Проверка демо

1. Убедись, что локальный WordPress запущен и ngrok работает (в терминале виден URL).
2. Открой бота в Telegram и нажми кнопку запуска Mini App (или команду `/start`, если там есть кнопка «Open app»).
3. Должен открыться твой фронт с Vercel; он отправит запрос на API_BASE_URL (ngrok). Если всё настроено, авторизация по Telegram пройдёт и откроется главный экран приложения.

Если что-то не работает — открой DevTools в браузере (если Mini App открывается в десктопном Telegram или по прямой ссылке) и посмотри ошибки в Network/Console; чаще всего это неверный или старый API_BASE_URL или недоступный ngrok.

---

## Краткий чеклист

| Шаг | Действие |
|-----|----------|
| 1 | WordPress локально запущен, плагин HR активен |
| 2 | ngrok установлен, `ngrok http 80` (или твой порт), скопирован https-URL |
| 3 | В WordPress в «Адрес сайта» временно указан ngrok-URL |
| 4 | В config.js фронта API_BASE_URL = ngrok-URL, деплой на Vercel |
| 5 | В BotFather создан бот, для него задан Mini App URL = Vercel-URL |
| 6 | Открываешь бота в Telegram → Open app → проверяешь демо |

---

## Важно про ngrok

- **Бесплатный ngrok** при каждом перезапуске даёт новый URL. После перезапуска нужно снова подставить новый URL в WordPress и в config.js и задеплоить фронт.
- Для демо на один-два дня можно не перезапускать ngrok — тогда ничего менять не нужно.
- Если нужен постоянный бесплатный хостинг под WordPress — можно позже перенести сайт на бесплатный хостинг (например InfinityFree, 000webhost) и в config.js указать уже постоянный URL вместо ngrok.
