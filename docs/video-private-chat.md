# Видео: приватный диалог по мэтчу

## Исправление в приложении

Вместо «HR service» везде отображается **HR Service CFO Club** (с большой буквы).

---

## Правильно ли я понял клиента

Да, смысл такой:

- **Club services** — общий экран «Сервисы клуба».
- Внутри него блок **HR Service CFO Club** — это именно HR-направление клуба (вакансии, кандидатуры, мэтчи, чаты). Не «роль HR», а название раздела сервисов.
- Под этим блоком — все пункты меню (Open vacancies, Place candidacy, My matches и для работодателя ещё My vacancies, Create vacancy, Candidates, Opened resumes). Для обычного участника — три пункта, для работодателя — плюс четыре.
- Идея «выпадажки» — блок HR можно свернуть/развернуть; позже можно добавить другие блоки (например, «Financial support»), тоже раскрывающиеся. Сейчас реализован один раскрывающийся блок «HR Service CFO Club».

Если клиент имел в виду другое (например, другое название или другую структуру) — можно уточнить и поправить.

## Пошагово: как записать видос и как создать приватный диалог

### Часть 1. Как создать мэтч (чтобы появился приватный диалог)

Приватный чат доступен **только по мэтчу**. Мэтч создаёт админ, потом его подтверждает работодатель.

1. **В WordPress (админ):**
   - Плагины → HR Ecosystem → **Match suggestions** (или раздел создания мэтчей).
   - Создать мэтч: выбрать вакансию и кандидата → **Create match**.
   - Открыть созданный мэтч в **Matches (HR)** → в поле **Status** выбрать **Pending employer** → **Update**.

2. **В мини-приложении под работодателем:**
   - Войти как пользователь с ролью Employer.
   - **Club services** → раскрыть **HR Service CFO Club** → **Candidates** (Pending my approval).
   - Найти нужный мэтч → **Approve**.

3. После этого мэтч станет **confirmed**. У **кандидата** он появится в **My matches**, у **работодателя** — в **Opened resumes**. У обоих появится кнопка **Chat** — это и есть вход в приватный диалог.

### Часть 2. Как записать видос

**Подготовка:**
- Два аккаунта: один **кандидат**, один **работодатель** (или один аккаунт + второй в другом браузере/телефоне).
- Один уже созданный и подтверждённый мэтч (как в части 1).
- Скринкаст с телефона (например, встроенная запись экрана в iOS/Android) или эмулятор + OBS/экранная запись на ПК.

**Сценарий записи:**

1. **Титры/озвучка (5–10 сек):**  
   «Показываю приватный диалог между кандидатом и работодателем по мэтчу и как в него попасть.»

2. **Кандидат (30–60 сек):**
   - Открыть мини-приложение под соискателем.
   - **Club services** → раскрыть **HR Service CFO Club**.
   - Нажать **My matches**.
   - В списке нажать **Chat** у нужного мэтча.
   - Показать экран чата: лента сообщений, поле ввода, кнопка Send. При желании написать тестовое сообщение и отправить.

3. **Работодатель (30–60 сек):**
   - Открыть приложение под работодателем (или переключить аккаунт).
   - **Club services** → **HR Service CFO Club** → **Opened resumes**.
   - Нажать **Chat** у того же кандидата/мэтча.
   - Показать тот же чат: те же сообщения, ответ работодателя, при необходимости кнопки **Make offer** / **Close dialog**.

4. **Опционально — WordPress (20–30 сек):**
   - Открыть админку WordPress → **Matches (HR)**.
   - Открыть этот мэтч.
   - Прокрутить вниз до метабокса **Chat (this match)** — показать историю переписки.

5. **Финальная фраза (5–10 сек):**  
   «Приватный диалог доступен только по мэтчу: кандидат заходит через My matches → Chat, работодатель — через Opened resumes → Chat. Переписка одна на мэтч и хранится на нашем сервере.»

### Часть 3. Как именно «создать» приватный диалог (кратко)

- **Не создаётся вручную отдельно.** Он появляется автоматически у мэтча после того, как:
  1. Админ создал мэтч и поставил статус **Pending employer**.
  2. Работодатель в приложении нажал **Approve** в **Candidates** (Pending my approval).
- После этого у кандидата в **My matches** и у работодателя в **Opened resumes** у этого мэтча есть кнопка **Chat** — нажатие открывает приватный диалог. Первое сообщение может написать любой из них.
